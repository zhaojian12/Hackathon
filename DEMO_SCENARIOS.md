## 🎬 完整演示流程

### 场景 1: 正常交易流程 ✅

**角色:** Alice (买家) 和 Bob (卖家)

#### 步骤 1: 风险评估

**Alice 准备购买 Bob 的 iPhone**

```
交易金额: 800 cUSD
交易描述: 购买二手 iPhone 14 Pro，9成新，有完整包装和配件
买家地址: 0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb
卖家地址: 0x5aAeb6053F3E94C9b9A09f33669435E7Ef1BeAed
```

**风险评估结果:**
```
✅ 风险评分: 18/100
✅ 风险等级: 低风险
✅ 建议: 继续交易
✅ 托管时长: 3 天
✅ 理由:
   - 金额适中
   - 描述详细清晰
   - 地址格式正常
   - AI 分析: 交易描述合理，无明显风险
```

#### 步骤 2: 创建交易

Alice 创建交易，资金进入智能合约托管

#### 步骤 3: 交易进行

- Bob 发货
- Alice 收货
- 双方确认

#### 步骤 4: 完成交易

资金自动释放给 Bob，交易完成 🎉

### 场景 2: 高风险交易被拦截 🚫

**角色:** Charlie (潜在受害者) 和 Scammer (诈骗者)

#### 步骤 1: 风险评估

**Charlie 看到一个"投资机会"**

```
交易金额: 50000 cUSD
交易描述: urgent investment opportunity guaranteed 100% profit bitcoin mining
买家地址: 0x1234567890abcdef1234567890abcdef12345678
卖家地址: 0x0000000000000000000000000000000000000001
```

**风险评估结果:**
```
🚨 风险评分: 92/100
🚨 风险等级: 高风险
🚨 建议: 不建议继续交易
🚨 托管时长: 14 天
🚨 理由:
   - 金额风险: 大额交易，需要特别注意
   - 检测到可疑关键词: urgent, investment, guaranteed, profit, bitcoin
   - 卖家地址: 可能是新地址
   - AI 分析: 存在明显诈骗特征，强烈建议不要交易
```

#### 结果

Charlie 看到警告后放弃交易，避免了损失 💰

### 场景 3: 争议仲裁 - 卖家未发货 ⚖️

**角色:** David (买家) 和 Eve (卖家)

#### 背景

David 购买了 Eve 的笔记本电脑，付款后 5 天卖家仍未发货

#### 步骤 1: 提交争议

**争议信息:**
```
交易金额: 3000 cUSD
交易描述: 购买联想笔记本电脑 ThinkPad X1
争议类型: 卖家未发货
买家主张: 已经付款 5 天了，卖家一直说马上发货，但没有任何物流信息
卖家回应: 最近比较忙，会尽快发货的
```

**聊天记录:**
```
Day 1:
David: 什么时候发货？
Eve: 今天就发

Day 3:
David: 还没发货吗？
Eve: 马上，马上

Day 5:
David: 已经 5 天了，请给物流单号
Eve: 再等等，很快的
```

**证据:**
- 买家证据:
  - 付款凭证截图
  - 聊天记录截图
  - 交易创建时间证明
- 卖家证据: (无)

#### 步骤 2: AI 仲裁分析

**仲裁结果:**
```
⚖️ 案件编号: CASE-20260206150000
⚖️ 责任方: 卖家责任
⚖️ 处理方案: 全额退款给买家
⚖️ 置信度: 88%

📋 详细理由:
1. 规则判断: 卖家未提供发货证明
2. 买家证据: ✅ 提供了文字说明; ✅ 提供了图片证据; ⚠️ 缺少物流信息
3. 卖家证据: ⚠️ 未提供证据; ⚠️ 缺少图片证据; ⚠️ 缺少物流信息
4. 聊天分析: 检测到拒绝沟通: 再等等
5. AI 分析: 卖家明显违约，多次承诺但未履行，建议全额退款

💡 操作建议:
- 置信度高，建议直接执行仲裁决定
```

#### 步骤 3: 执行仲裁

平台审核后，执行全额退款，David 收回资金 ✅

### 场景 4: 争议仲裁 - 商品损坏 📦

**角色:** Frank (买家) 和 Grace (卖家)

#### 背景

Frank 收到的商品有损坏，双方对责任有争议

#### 步骤 1: 提交争议

**争议信息:**
```
交易金额: 1500 cUSD
交易描述: 购买机械键盘 Cherry MX
争议类型: 商品损坏
买家主张: 收到的键盘有多个按键失灵，包装有破损痕迹
卖家回应: 发货时检查过，完好无损，可能是快递的问题
```

**聊天记录:**
```
Frank: 收到货了，但是有几个键不能用
Grace: 不可能，我发货前测试过的
Frank: 我有开箱视频
Grace: 那可能是快递摔坏的，不是我的责任
Frank: 但是你没有保价
Grace: 这个...我们协商一下吧
```

**证据:**
- 买家证据:
  - 键盘损坏照片（3张）
  - 包装破损照片
  - 开箱视频
- 卖家证据:
  - 发货前测试视频
  - 物流单号: SF1234567890

#### 步骤 2: AI 仲裁分析

**仲裁结果:**
```
⚖️ 案件编号: CASE-20260206151500
⚖️ 责任方: 卖家责任
⚖️ 处理方案: 部分退款
⚖️ 置信度: 72%

📋 详细理由:
1. 规则判断: 买家提供了损坏证据，建议部分退款
2. 买家证据: ✅ 提供了图片证据; ✅ 提供了文字说明; ⚠️ 缺少物流信息
3. 卖家证据: ✅ 提供了图片证据; ✅ 提供了物流信息
4. 聊天分析: 发现承诺性语言: 协商
5. AI 分析: 双方都有证据，商品确实损坏，但无法确定是发货前还是运输中。
   考虑到卖家未保价，建议卖家承担部分责任，退款 50-70%

💡 操作建议:
- 置信度中等，建议人工复核后执行
- 建议退款比例: 30-70%
- 可协商买家退货后退款
```

#### 步骤 3: 协商解决

平台建议退款 60%（900 cUSD），双方接受，争议解决 🤝

### 场景 5: 智能客服咨询 💬

**用户:** Helen (新用户)

#### 对话 1: 如何创建交易

```
Helen: 我想买东西，怎么创建交易？

AI: 创建交易很简单，按以下步骤操作：

1. 连接钱包
   - 点击右上角"Connect Wallet"
   - 选择你的钱包（MetaMask/OKX/Fluent）
   - 授权连接

2. 填写交易信息
   - 输入卖家地址
   - 选择代币类型（CFX 或 cUSD）
   - 输入交易金额
   - 添加交易描述（可选）

3. 创建交易
   - 点击"创建交易"按钮
   - 在钱包中确认交易
   - 等待区块链确认（约 3-10 秒）

资金会自动托管在智能合约中，等待卖家确认。

需要我详细解释某个步骤吗？
```

#### 对话 2: 风险评估

```
Helen: 什么是风险评估？

AI: 风险评估是我们的智能安全功能，可以在交易前帮你分析潜在风险：

功能特点：
- 分析交易金额是否异常
- 检测诈骗关键词
- 验证地址格式
- AI 深度分析

使用方法：
1. 点击顶部导航"风险评估"
2. 填写交易信息
3. 点击"分析风险"
4. 查看风险报告

建议：
- 大额交易前必评估
- 高风险交易谨慎进行
- 关注 AI 给出的具体原因

这个功能可以帮你避免很多诈骗陷阱！
```

#### 对话 3: 争议处理

```
Helen: 如果卖家不发货怎么办？

AI: 如果遇到卖家不发货的情况，可以通过争议仲裁解决：

处理流程：
1. 提交争议
   - 点击"争议仲裁"页面
   - 选择争议类型"卖家未发货"
   - 填写详细情况

2. 提供证据
   - 付款凭证
   - 聊天记录
   - 其他相关证据

3. AI 分析
   - 系统会自动分析
   - 给出责任判定
   - 提供处理建议

4. 平台审核
   - 人工复核 AI 建议
   - 执行仲裁决定
   - 资金退回或放款

保护措施：
- 资金托管在智能合约
- 卖家无法直接提取
- 平台公正仲裁

放心交易，我们会保护你的权益！
```

### 风险评估演示

1. **展示低风险交易**
